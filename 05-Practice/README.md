# 05 - Practice

## Why this folder exists

Knowledge without practice is temporary.

The purpose of this folder is to convert understanding into instinct through consistent problem solving.

Practice is not about solving the most difficult question.

It is about solving questions correctly, explaining the reasoning, and recognizing patterns.

---

## Practice Strategy

Always solve problems using the following order:

1. Read the question carefully.
2. Identify the input and output.
3. Find the keywords.
4. Recognize the pattern.
5. Choose the data structure.
6. Choose the JavaScript feature.
7. Write the algorithm.
8. Write the code.
9. Dry run the solution.
10. Analyze time and space complexity.

---

## Difficulty Levels

### Easy

Focus on one concept at a time.

Examples:

- Variables
- Loops
- Arrays
- Objects

---

### Medium

Combine multiple concepts.

Examples:

- Arrays + Objects
- Loops + Functions
- Set + Map

---

### Hard

Require strong problem-solving skills and multiple patterns.

Examples:

- Sliding Window
- Recursion
- Dynamic combinations
- Advanced interview questions

---

## Goal

Do not solve the most questions.

Solve every question with a clear understanding of:

- Why this approach works.
- Why another approach was rejected.
- Which pattern was recognized.
- Which JavaScript methods were selected.
