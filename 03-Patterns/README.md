# 03 - Problem Solving Patterns

## Why Patterns Matter

Most programming problems are not completely new.

They are variations of problems that have been solved many times before.

Instead of memorizing solutions, experienced developers recognize patterns.

Once the pattern is identified, choosing the correct algorithm, data structure, and JavaScript methods becomes much easier.

The goal of this section is to train pattern recognition.

---

## What is a Pattern?

A pattern is a repeatable way of solving a particular type of problem.

Examples:

Need to visit every item?

→ Iteration

Need to find duplicates?

→ Lookup

Need to count frequencies?

→ Counting

Need to keep only matching items?

→ Filtering

Need to transform every item?

→ Transformation

Need one final result?

→ Aggregation

Need two indexes moving together?

→ Two Pointers

Need a moving range?

→ Sliding Window

Need to solve a problem by calling itself?

→ Recursion

---

## Why Learn Patterns?

When solving interview questions, don't ask:

"What method should I use?"

Instead ask:

"What kind of problem is this?"

The answer to that question determines the solution.

## Goal

By the end of this section, you should be able to recognize the underlying pattern of a problem before writing any code.

Choosing the correct pattern should become an automatic habit.
