# Problem Solving Framework

## Why does this framework exist?

Programming is not about remembering syntax.

Programming is about understanding problems and making good decisions.

Every programming problem contains clues.

Those clues help us identify:

- What is changing?
- What stays the same?
- Which data structure fits?
- Which algorithm fits?
- Which JavaScript feature fits?
- Which method naturally solves the problem?

The purpose of this framework is to build a repeatable thinking process that works for every programming problem.

---

# The Golden Rule

Never write code immediately.

Understand first.

Design second.

Code last.

# Philosophy

We are not learning JavaScript.

We are learning how to solve problems using JavaScript.

Syntax is a tool.

Thinking is the skill.
